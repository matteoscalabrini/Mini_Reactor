# Port Plan — ESP-IDF 5.5.x / Arduino-ESP32 3.x

**Branch:** `feature/idf55-port`
**Date:** 2026-06-25
**Driver:** Claude.md mandate — *"The firmware must rely on Idf 5.5.x and relative arduino version."*
**Current state:** `espressif32@6.9.0` → Arduino-ESP32 **2.0.17** → ESP-IDF **4.4.7** (verified via core `esp_idf_version.h`).
**Target:** pioarduino platform → Arduino-ESP32 **3.x** → ESP-IDF **5.5.x**.

> Scope-only document. No source changed yet. Tasks are ordered to keep the tree
> buildable at each checkpoint and follow Claude.md's incremental-change rule.

---

## 1. Platform swap (`platformio.ini`)

The official `espressif32` platform tops out at Arduino 2.x / IDF 4.4. IDF 5.5
requires the community **pioarduino** fork.

- [ ] Replace `platform = espressif32@6.9.0` with the pioarduino release that ships **IDF 5.5.x**.
  - pioarduino release scheme: `55.03.xx-x` → IDF 5.5. **VERIFY exact tag** before pinning:
    `platform = https://github.com/pioarduino/platform-espressif32/releases/download/55.03.xx/platform-espressif32.zip`
- [ ] Keep `board = esp32-s3-devkitc-1`, `framework = arduino`.
- [ ] Confirm `board_build.partitions = partitions.csv` and `board_build.filesystem = spiffs` still honored (SPIFFS is deprecated-but-present in 3.x).
- [ ] Re-verify `build_flags`: `-DARDUINO_USB_CDC_ON_BOOT=1` (valid), `-DASYNCWEBSERVER_REGEX=1` (mathieucarbou fork still reads this).

## 2. Code breaks (confirmed by source scan)

### 2.1 LEDC — heater PWM (DEFINITE break)
`src/heater/Heater.cpp` uses the **channel-based** LEDC API removed in 3.x.
- [ ] `ledcSetup(channel, freq, bits)` + `ledcAttachPin(pin, channel)` → `ledcAttach(pin, freq, resolution)` (pin-based, returns bool).
- [ ] `ledcWrite(channel, duty)` → `ledcWrite(pin, duty)`.
- [ ] Drop `AppConfig::Heater::kLedcChannel` and `Config::ledcChannel` (channels auto-assigned in 3.x). Update [app_config.hpp](../../../include/app_config.hpp) + `makeHeaterConfig()`.
- [ ] Check `ledcAttach` return value in `begin()`, log success/failure (fits toggle-contract startup-log rule).

### 2.2 ADC — thermistor (LOW risk, verify)
`src/sensor/Thermistor.cpp`: `analogReadResolution`, `analogSetPinAttenuation(pin, ADC_11db)`, `analogReadMilliVolts`, `analogRead` all survive in 3.x.
- [ ] Confirm `ADC_11db` not removed (may emit deprecation → consider `ADC_ATTENDB_MAX`).
- [ ] Sanity-check `analogReadMilliVolts` calibration values vs 4.4.7 (ADC oneshot driver changed underneath).

### 2.3 SPIFFS — web UI (LOW risk)
`src/net/WebInterface.cpp` uses `SPIFFS`. Present in 3.x but deprecated.
- [ ] Keep SPIFFS for the port (no behavior change). Track LittleFS migration as **separate** backlog item — not part of this port.

## 4. Build & verification sequence

- [ ] Clean build dir (core major bump): `rm -rf .pio` then `~/.platformio/penv/bin/pio run -e esp32-s3-devkitc-1`.
- [ ] Resolve compile errors in dependency order: heater LEDC → libs → link.
- [ ] **Full erase + reflash** (treat core bump like a partition change per Claude.md): erase flash, flash firmware, re-`uploadfs` the SPIFFS UI.
- [ ] Hardware smoke test (see [memory: hardware-validation-setup]): boot log, get IP, `GET /api/v1/status`, heater duty sweep, motor jog, DS18B20 read, OLED render.
- [ ] Confirm reported IDF version is 5.5.x at boot (add a boot-log line if absent).
